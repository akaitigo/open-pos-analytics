# RFM セグメンテーション定義（cohort モジュール / Issue #11）

顧客を **優良 / 離脱リスク / 休眠** の3セグメントに分類するルールと、その根拠を定義する。
本ドキュメントが分類ロジックの一次情報であり、コード（`CohortAggregationService`・`V4__rfm_segments.sql`）はこの定義の実装である。

## 対象母集団

`customers` テーブルの全会員（`customer_hash` を持つ取引から生成される）。
会員が0件の場合は「会員データなし」として API は 200 でフォールバック応答を返し、分類は行わない（PRD 技術リスク2 / ADR-0003）。

生の会員ID・氏名・連絡先は保持しない。分析はハッシュ（`customer_hash`）のみで行い、API・UI にハッシュ値そのものも出さない。

## 基準日（as-of date）

`as_of = MAX(last_seen_at)`（母集団の最終来店日の最大値）。

サンプル/履歴データは実時刻の「今」ではないため、実行日を基準にすると全員が離脱扱いになる。
データ内で観測された最新の活動時点を「今」とみなすことで、Recency を相対評価する。

## 特徴量（生値）

| 記号 | 定義 | 出典カラム | 良い方向 |
|------|------|-----------|---------|
| R (recency_days) | `as_of::date - last_seen_at::date`（日数、0以上） | `customers.last_seen_at` | 小さいほど良い（最近来た） |
| F (frequency) | 来店回数 = 取引件数 | `customers.visit_count` | 大きいほど良い |
| M (monetary) | 累計購入金額 | `customers.total_spent` | 大きいほど良い |

## スコアリング（3分位 / tertile ランク）

R・F・M それぞれを母集団内で **3分位（`NTILE(3)`、3等分ランク）** によりスコア化する。
**スコア3が最良**、スコア1が最下位。決定性のため同値は `customer_hash` の昇順で分割する。

| スコア | r_score（Recency） | f_score（Frequency） | m_score（Monetary） |
|-------|---------------------|----------------------|----------------------|
| 3 | 最も最近に来店した上位1/3 | 来店回数が上位1/3 | 購入金額が上位1/3 |
| 2 | 中位1/3 | 中位1/3 | 中位1/3 |
| 1 | 最も来店が古い下位1/3 | 来店回数が下位1/3 | 購入金額が下位1/3 |

`NTILE(3)` は件数を3等分するランクベースの分位であり、値の絶対閾値ではなく順位で区切る。
少数母集団（例: 会員2名）でも破綻せず、分布の偏りに頑健なため採用する。

r_score は `last_seen_at` 昇順（古い→新しい）の `NTILE(3)` で求める。
昇順の先頭（最も古い＝最下位）に 1、末尾（最も新しい＝最良）に 3 が割り当たる。

## セグメント分類ルール（MECE）

複合価値 `V = f_score + m_score`（範囲 2〜6）と Recency で分類する。全会員が排他的に1セグメントへ入る。

| セグメント | id | 条件 | 意味 |
|-----------|----|----|------|
| 優良 | `loyal` | `V >= 4` かつ `r_score >= 2` | 頻度・金額の価値が高く、最近も来店がある中核顧客 |
| 離脱リスク | `at_risk` | `V >= 4` かつ `r_score == 1` | かつては高頻度・高金額だが、最近の来店が最下位1/3に落ちた顧客。**販促の最優先ターゲット**（PRD ユーザーストーリー4） |
| 休眠 | `dormant` | `V <= 3` | 頻度・金額の複合価値が低い低活性顧客 |

### 根拠

- **Recency で優良と離脱リスクを分ける**: 高価値顧客（`V >= 4`）のうち、まだ来ている層（`r_score >= 2`）が優良、来なくなった層（`r_score == 1`）が離脱リスク。「離脱前に販促を打つ」という業務要求（ユーザーストーリー4）に直結する。
- **複合価値 V で休眠を切り出す**: 頻度・金額がともに低い層（`V <= 3`）は、育成対象の低活性顧客として休眠に集約する。

### 既知の単純化（MVP）

- 「最近来たが低価値」の顧客（例: `r_score=3, f_score=1, m_score=1`）は `V<=3` により **休眠** に入る。厳密には新規/一見客だが、MVP では「まだ関係が定着していない育成対象」として休眠プールにまとめる。Phase 1 で「新規」セグメントの分離を検討する。
- スコアは母集団相対（3等分）であり、絶対的な優良基準ではない。母集団が小さい/偏っている場合、各セグメントは常に概ね等サイズにならず、`V` と Recency の分布に依存する。

## 簡易 LTV 推定

PRD の通り「セグメント平均 × 残存期間の簡易版」とする（Phase 1 で精緻化）。

- 平均客単価 `avgOrderValue = Σmonetary / Σfrequency`（セグメント内の総購入額 ÷ 総来店回数）
- 推定LTV `estimatedLtv = avgOrderValue × 想定残存来店回数`

想定残存来店回数はセグメント別の固定ヒューリスティック（優良=24 / 離脱リスク=8 / 休眠=4）。
これは将来価値の粗い当たりであり、確度の高い推定ではない旨を UI にも明示する。

## API との対応

- `GET /api/rfm`（任意で `?segment=loyal|at_risk|dormant`）はセグメント別に
  件数・平均客単価・平均Recency・最終来店日・推定LTV を返す。
- `segment` に enum 以外を渡すと 400。
- 会員0件時は `hasMemberData=false` の 200 応答（UI はフォールバック案内）。
