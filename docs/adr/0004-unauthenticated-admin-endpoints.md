# ADR-0004: 管理エンドポイント（取込・再集計）は MVP では無認証とする

- **日付**: 2026-07-12
- **ステータス**: 承認済み
- **決定者**: Ryusei

## コンテキスト

CSV取込（IngestService）と集計再計算（3つの AggregationService）に、テスト以外から到達する手段が無く、クイックスタートが成立しなかった。最小の運用入口として `POST /api/admin/ingest` / `POST /api/admin/recompute` を追加する。

セキュリティルール上、認証が必要なエンドポイントには認証を付けるのが原則であり、MVP で対象外とする場合は ADR に明記する必要がある。

## 決定

MVP では管理エンドポイントを**無認証**で提供する。ただし以下の前提と制約を課す。

1. **ローカル運用前提**: 本プロダクトの MVP は店主のローカル環境（localhost）での利用を想定し、公開ネットワークへのデプロイは行わない
2. **入力制約**: path は絶対パス必須・`.csv` 拡張子のみ・実在する通常ファイルのみ・100MB 上限・4096文字上限。テーブル名は許可リスト固定で SQL は文字列結合しない（COUNT のみ）
3. **本番化の条件**: Cloud Run 等へデプロイする際は、認証（IAM/OIDC）を必須とし、`/api/admin/*` をエッジで遮断するか認可フィルタを実装するまで公開しない

## 却下した選択肢

- **Gradle タスクからの直接取込**: IngestService は Quarkus コンテナ（CDI/JPA）を要するため、JavaExec からの単独起動は成立しない
- **dev プロファイル限定エンドポイント**: デモ・検証で prod ビルドでも取込が必要になるため、限定はせず ADR で運用前提を明示する方式を採る
