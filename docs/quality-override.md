# 品質チェックリスト追加分 — open-pos プラグイン

> Layer-0（共通）+ Layer-1（言語別）のチェックリストに**追加**する項目のみ。

## open-pos プラグイン固有の品質基準

### PCI-DSS セキュリティ要件
- [ ] カード番号（PAN）がログに出力されない
- [ ] CVV/CVC がメモリ上で不要になった時点で破棄される
- [ ] 機密データが localStorage / sessionStorage に保存されていない
- [ ] 通信は全て TLS 1.2+ で暗号化されている
- [ ] eval() / Function() コンストラクタが使用されていない
- [ ] 依存パッケージに既知の脆弱性がない（`npm audit` / `yarn audit`）

### open-pos 互換性
- [ ] `manifest.json` がプラグイン仕様に準拠している
- [ ] `@open-pos/sdk` の最小バージョン要件を満たしている
- [ ] プラグインのライフサイクルフック（init, destroy）が実装されている
- [ ] open-pos のイベントバスとの連携が正しく動作する
- [ ] 他のプラグインとの名前空間の衝突がない

### API連携
- [ ] open-pos API のバージョニングに対応している
- [ ] API呼び出しにリトライ機構がある
- [ ] タイムアウトが設定されている
- [ ] エラーハンドリングがユーザーに適切なフィードバックを返す

### テスト
- [ ] open-pos テストハーネスでの統合テストが通過する
- [ ] プラグイン単体での動作テストがある
- [ ] モック化されたopen-pos環境でのテストが可能

### UI/UX（UIプラグインの場合）
- [ ] open-pos のデザインシステムに準拠している
- [ ] 多言語対応（i18n）のフレームワークを使用している
- [ ] レスポンシブ対応（POS端末のサイズバリエーション）
